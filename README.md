# finex — Virtual Screening Pipeline for Long COVID Drug Repositioning

## Context

Zhang et al. (PNAS 2026) demonstrate that proteolytic digestion of the SARS-CoV-2 Spike protein generates toxic peptide fragments called **xenoAMPs**. These fragments induce Negative Gaussian Curvature (NGC) on cell membranes, forming transmembrane pores that selectively destroy immune cells with complex morphologies: plasmacytoid dendritic cells (pDC), CD4+ and CD8+ T lymphocytes. The collapse of CD8+ populations prevents control of latent viruses (EBV, CMV) and drives chronic hyperinflammation characteristic of severe Long COVID.

The paper identifies **xenoAMP(S)** — residues 529–558 of the Spike protein (`KSTNLVKNKCVNFNFNGLTGTGVLTESNKK`) — as the primary active fragment, and proposes that small molecules capable of binding and sterically blocking this peptide could prevent pore formation.

**finex** implements this proof of concept as an automated virtual screening pipeline: it docks FDA-approved drugs and natural compounds against the xenoAMP(S) structure to identify candidate inhibitors.

## Installation

**Prerequisites:** Python ≥ 3.12, [OpenBabel](https://openbabel.org) and [AutoDock Vina](https://vina.scripps.edu) in `PATH`.

```bash
# macOS
brew install open-babel autodock-vina

# Install Python dependencies
pip install uv
uv sync
```

## Data Directory Structure

The `data/` directory is not versioned (too large). Create it manually with the following structure:

```
data/
├── receptor_xenoAMP.pdb          # 3D structure of xenoAMP(S) — generated by `run`
├── receptor_xenoAMP.pdbqt        # PDBQT version — generated by `run`
├── ligands_in/                   # Input SDF files (FDA drugs + supplements)
├── ligands_pdbqt/                # Converted PDBQT files — generated by `run`
├── docking_results/              # Vina output files — generated by `run`
└── supplements_cache/
    ├── cmaup/                    # CMAUP v2.0 files (download manually — see below)
    │   ├── CMAUPv2.0_download_Plants.txt
    │   ├── CMAUPv2.0_download_Ingredients_onlyActive.txt
    │   └── CMAUPv2.0_download_Plant_Ingredient_Associations_onlyActiveIngredients.txt
    └── foodb/                    # FooDB CSV files (download manually — see below)
        ├── Food.csv
        ├── Compound.csv
        └── Content.csv
```

### Downloading supplement databases

**CMAUP v2.0** — [bidd.group/CMAUP](https://bidd.group/CMAUP/download.html)
Download and place in `data/supplements_cache/cmaup/`:
- `CMAUPv2.0_download_Plants.txt`
- `CMAUPv2.0_download_Ingredients_onlyActive.txt`
- `CMAUPv2.0_download_Plant_Ingredient_Associations_onlyActiveIngredients.txt`

**FooDB** — [foodb.ca/downloads](https://foodb.ca/downloads)
Download the CSV export, extract and place in `data/supplements_cache/foodb/`:
- `Food.csv`, `Compound.csv`, `Content.csv`

## Usage

### Full pipeline (FDA-approved drugs)

```bash
# Run complete screening — downloads drugs from ChEMBL automatically
python finex.py run

# Resume after interruption (skips already-docked molecules)
python finex.py run

# Force re-download of FDA database
python finex.py run --fresh-db
```

### Screening natural compounds and food supplements

```bash
# Generate 3D structures from all available databases (CMAUP + FooDB)
python finex.py fetch-supplements

# Filter by specific plants (CMAUP)
python finex.py fetch-supplements --source cmaup --filter "curcuma longa,ginkgo biloba,withania somnifera"

# Filter by food (FooDB)
python finex.py fetch-supplements --source foodb --filter "turmeric,garlic,green tea"

# List available plants / foods
python finex.py fetch-supplements --list-plants
python finex.py fetch-supplements --list-foods

# Then run docking on generated structures
python finex.py run
```

### Reporting (check progress mid-run)

```bash
# Rebuild Excel ranking with all docked molecules so far
python finex.py report

# Show top 50
python finex.py report --top 50
```

### Refinement of top candidates

```bash
# Re-dock top 20 with higher exhaustiveness for better accuracy
python finex.py refine --top 20 --exhaustiveness 64
```

### Visualization

```bash
# Generate PDB complex files for PyMOL (top 5 by default)
python finex.py viz --top 10
pymol output/viz/*_complex.pdb
```

## Docking Parameters

| Parameter | Default | Notes |
|-----------|---------|-------|
| `--exhaustiveness` | 8 | Initial screening. Use 32–64 for refinement. |
| `--max-torsions` | 15 | Molecules with more rotatable bonds are excluded |
| Grid center | (-1.0, 5.0, -2.5) Å | Centered on xenoAMP(S) hydrophilic insertion region |
| Grid size | 20 × 20 × 20 Å | Covers the full membrane-insertion face |

---

## Recommended Workflow

```bash
# 1. Generate all ligands (FDA + supplements)
python finex.py fetch-supplements
python finex.py run                    # run nightly, resumes automatically

# 2. Check progress each morning
python finex.py report

# 3. Once complete, refine top candidates
python finex.py refine --top 20 --exhaustiveness 64

# 4. Visualize
python finex.py viz --top 10
```
